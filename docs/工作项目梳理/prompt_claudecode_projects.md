# Claude Code 项目探索 Prompt
# 用途：粘贴进 Claude Code，让它并行探索 4 个项目，输出简历素材

---

## 使用说明

直接把下面的内容粘贴到 Claude Code 对话框，回车执行。
Claude Code 会用 SubAgents 并行探索 4 个项目目录。

---

## Prompt 正文

```
我需要你帮我为求职准备简历素材。目标岗位是 AI Agent 开发工程师，我是全栈开发背景，有一年工作经验。

请使用 SubAgents 并行探索以下 4 个项目目录，每个 SubAgent 负责一个项目：

项目 1：YDC 非标清单数据分析
路径：/Users/stephen/Downloads/00_project/13_YOMY相关/YDC 数据分析可视化

项目 2：公司官网
路径：/Users/stephen/Downloads/00_project/12_YOMY官网

项目 3：爬虫招标信息抓取
路径：/Users/stephen/Downloads/00_project/13_YOMY相关/招标信息自动化抓取

项目 4：清单与 Revit 族信息管理系统
路径：/Users/stephen/Downloads/00_project/13_YOMY相关/YBP-系统设计

---

每个 SubAgent 探索项目时，请按以下顺序读取文件：
1. 先读 README 或任何说明文档（如果有）
2. 读项目根目录结构（tree 或 ls -la）
3. 读前端入口文件和核心页面组件
4. 读后端入口文件和核心 API 路由
5. 搜索所有包含 AI/LLM/Claude/OpenAI/FastAPI/agent/prompt 的文件
6. 读 package.json / requirements.txt / pom.xml 了解依赖

---

每个项目探索完成后，按以下模板输出，不要省略任何字段：

### [项目名称]

**1. 这个项目是干什么的（一句话）**
用产品视角描述：解决了谁的什么问题

**2. 我负责的具体模块**
列出所有功能模块，标注哪些是独立设计、哪些是主导开发的

**3. 技术栈（越具体越好）**
- 前端：框架、UI库、状态管理、路由等
- 后端：语言、框架、数据库、ORM等
- AI/LLM：用了哪些 API、哪些框架、哪些模型
- 部署/工具：Docker、CI/CD、云服务等

**4. AI 和自动化相关（重点）**
- 哪些功能用到了 AI/LLM/爬虫/自动化
- 具体是怎么实现的（调用哪个 API、用了什么 Prompt 策略、怎么处理输出）
- 和 Agent/工具调用/知识库最相关的部分是什么

**5. 可量化的数据（从代码和配置中推断）**
- 处理的数据规模（行数、文件数、接口数）
- 有没有批处理、分页、并发相关的代码
- 任何能体现规模或效果的数字

**6. 产品设计决策（从代码结构推断）**
- 为什么选这个技术栈而不是其他
- 数据流怎么设计的
- 有没有体现产品思维的设计（错误处理、用户体验、可扩展性）

**7. 简历可用的一句话项目描述**
格式：用 [技术栈] 开发了 [功能描述]，解决了 [具体问题]，实现了 [可量化成果]

---

4 个项目全部探索完后，额外输出一个汇总：

## 跨项目技术汇总

**核心技术栈（所有项目共有）**

**差异化亮点（只在某个项目出现的特色技术或设计）**

**AI/Agent 相关能力总结（最适合写进 AI Agent 岗位简历的内容）**

**最适合作为简历重点项目的排序（1-4名，说明理由）**
```
